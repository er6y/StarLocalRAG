# StarLocalRAG 使用说明

## 1. RAG 基本原理介绍

RAG（检索增强生成）是一种结合了检索系统和生成式AI的技术框架，能够让大语言模型(LLM)基于特定知识回答问题，提高回答的准确性和可靠性。

### RAG 工作流程

```
+----------------+     +----------------+     +----------------+
|                |     |                |     |                |
|  用户提问       | --> |  文本向量化     | --> |  知识库检索     |
|                |     |                |     |                |
+----------------+     +----------------+     +----------------+
                                                     |
                                                     v
+----------------+     +----------------+     +----------------+
|                |     |                |     |                |
|  生成回答       | <-- |   大语言模型    | <-- |  构建上下文     |
|                |     |   (LLM)        |     |                |
+----------------+     +----------------+     +----------------+
```

### RAG 核心步骤

1. **知识库构建**：
   - 导入文档（PDF、Word、文本等）
   - 文本分块处理
   - 使用嵌入模型将文本块转换为向量
   - 存储文本及其向量到向量数据库

2. **问答交互**：
   - 用户提出问题
   - 问题向量化
   - 在向量数据库中检索相似文本块
   - 将检索结果与问题一起发送给大语言模型
   - 大语言模型生成基于知识库的回答

3. **优势**：
   - 回答基于特定知识，减少"幻觉"
   - 无需重新训练模型即可更新知识
   - 保持私密性，数据不离开本地设备

### 文本分块策略

#### 1. 暴力分块方法
- **定义**：将文档按固定长度(如500字符)简单切分
- **优势**：
  - 实现简单，计算成本低
  - 适合内容结构简单的文档
- **缺陷**：
  - 容易切断语义连贯性
  - 检索结果可能包含无关内容
  - 对LLM归纳能力要求高

#### 2. 检索结果分析
- **有用信息**：与问题直接相关的文本片段
- **噪声信息**：
  - 同一文本块中不相关内容
  - 格式标记、页眉页脚等
- **对LLM的要求**：
  - 需要从混杂内容中提取关键信息
  - 必须识别并忽略无关内容
  - 需保持原始语义的准确性

#### 3. 推荐分块策略
| 策略 | 适用场景 | 示例参数 | 优点 |
|------|---------|---------|------|
| 固定长度分块 | 技术文档/代码 | 500字符，100重叠 | 简单高效 |
| 语义分块 | 叙述性内容 | 按段落/章节 | 保持语义完整 |
| 层次分块 | 结构化文档 | 标题+内容组合 | 保留上下文 |
| 动态分块 | 混合内容 | 基于内容分析 | 自适应最佳 |

**最佳实践建议**：
1. 技术文档：500-1000字符分块，20%重叠
2. 叙述文本：按自然段落分块
3. 混合内容：先按结构分块，再对长块二次分块
4. 添加元数据标记块类型和关键字段

## 2. StarLocalRAG 软件介绍

StarLocalRAG 是一款安卓本地 RAG 应用，允许用户在设备本地构建知识库、进行问答交互和管理知识笔记，无需依赖云服务。应用支持多种文档格式，使用嵌入模型进行向量化，并通过本地数据库进行高效检索。

### 主要特点

- **完全本地化**：所有处理和存储都在设备本地进行，保护数据隐私
- **多格式支持**：支持PDF、Word、Excel、PPT、文本等多种文档格式
- **知识笔记**：支持创建和管理知识笔记，方便整理和扩充知识库
- **灵活配置**：支持自定义嵌入模型、分块大小、API设置等
- **性能优化**：优化内存使用和处理速度，支持大文件处理

## 3. 使用说明

### 3.1 主界面导航

StarLocalRAG 应用包含三个主要页面，通过底部导航栏进行切换：

- **RAG问答**：进行基于知识库的问答交互
- **构建知识库**：创建和更新知识库
- **知识笔记**：创建和管理知识笔记

### 3.2 RAG问答页面

#### 界面元素说明

- **API URL下拉框**：选择或输入LLM服务的API地址
- **API密钥输入框**：输入API密钥
- **模型选择下拉框**：选择使用的LLM模型
- **知识库下拉框**：选择要查询的知识库
- **系统提示词输入框**：设置发送给LLM的系统提示词
- **用户问题输入框**：输入要询问的问题
- **回答显示区域**：显示AI回答，支持Markdown渲染
- **发送/停止按钮**：发送问题或停止生成
- **近似深度设置**：控制向量数据库检索后提交给模型的文本块数量
  - 值越大，提交的上下文越多，回答可能更准确但速度更慢、消耗更多token
  - 值越小，响应更快但可能遗漏相关信息
  - 建议根据需求平衡：一般问答3-5，精确检索可设8-10

#### 操作说明

1. 选择API设置和知识库
2. 输入系统提示词（可选）
3. 在问题输入框中输入问题
4. 点击发送按钮获取回答
5. 可以通过长按回答文本进行复制或转为笔记
6. 点击停止按钮可以中断生成过程

### 3.3 构建知识库页面

#### 界面元素说明

- **知识库名称下拉框**：选择或创建知识库
- **嵌入模型下拉框**：选择用于生成文本向量的模型
- **文件列表显示区域**：显示已选择的文件
- **进度显示区域**：显示处理进度
- **浏览文件按钮**：选择要添加到知识库的文件
- **创建知识库按钮**：开始处理文件并构建知识库

#### JSON文件特殊处理
1. **自动识别格式**：
   - Alpaca格式（instruction/input/output）
   - DPO格式（prompt/chosen）
   - 对话格式（conversations/messages）

2. **处理规则**：
   - 最小文本块大小：使用设置中配置的值（默认50字符）
   - 大文件(>1MB)自动分片
   - 保留语义完整性

3. **示例**：
   ```json
   // Alpaca格式
   {"instruction":"解释AI","output":"人工智能是..."}
   
   // DPO格式
   {"prompt":"机器学习定义","chosen":"机器学习是..."}
   ```

#### 操作说明

1. 输入或选择知识库名称
2. 选择嵌入模型
3. 点击"浏览文件"按钮选择文件
4. 文件会显示在列表中
5. 点击"创建知识库"按钮开始处理
6. 处理过程中会显示进度，建议把程序留在前台不要手动熄屏。程序在构建的时候回防止自动锁屏。

### 3.4 知识笔记页面

#### 界面元素说明

- **知识库下拉框**：选择要添加笔记的知识库
- **标题输入框**：输入笔记标题
- **内容输入框**：输入笔记内容
- **添加到知识库按钮**：将笔记添加到知识库
- **进度显示区域**：显示处理进度
  - 处理中：显示当前处理进度百分比
  - 成功后会显示向量数据库项目数量变化（如"项目数：100 → 101"表示新增1条）
  - 失败时会显示错误信息

#### 操作说明

1. 选择知识库
2. 输入笔记标题和内容
3. 点击"添加到知识库"按钮
4. 处理完成后会显示成功信息
5. 也可以从RAG问答页面通过长按回答文本，选择"转笔记"功能快速创建笔记

### 3.5 设置页面

#### 界面元素说明

1. **基本设置**
   - 模型目录：本地LLM的目录路径
   - 嵌入模型目录：设置存储嵌入模型的目录路径
   - 知识库目录：设置存储知识库的目录路径

2. **文本处理设置**
   - 分块大小：设置文本分块的大小（默认1000字符）
   - 重叠大小：设置文本分块的重叠大小（默认200字符）
   - 最小分块限制：设置文本分块的最小长度（默认50字符）
   - JSON训练集分块：特殊处理JSON格式的训练数据集

3. **性能设置**
   - GPU加速：启用GPU加速处理（需设备支持）

4. **显示设置**
   - 字体大小：调整应用中文本的显示大小

5. **调试设置**
   - 调试模式：启用详细日志输出

#### 操作说明

1. 从主菜单点击"设置"进入设置页面
2. 调整各项设置
3. 设置会自动保存
4. 返回主界面时，设置会立即生效

### 3.6 菜单功能

StarLocalRAG 应用在右上角提供了菜单选项：

- **设置**：打开设置页面，配置应用参数
- **关于**：显示应用版本信息
- **查看日志**：打开日志查看页面，查看应用运行日志
  - 支持点击选择单条或多条日志
  - 长按可复制选中日志内容
  - 支持通过分享功能转发选中日志
- **帮助**：打开本使用说明
- **退出**：关闭应用

### 3.7 日志查看页面

#### 界面元素说明

- **日志显示区域**：显示应用运行日志
- **清空日志按钮**：清除当前日志
- **刷新按钮**：刷新日志内容
- **返回按钮**：返回主界面

#### 日志菜单选项


- **清空日志**：清除所有日志
- **日志分享功能**：
  - 支持导出完整日志文件(.log格式)
  - 可通过系统分享菜单发送到微信等社交平台
  - 可选择分享全部日志或仅选中日志
  - 分享时会自动附加设备型号和系统版本信息，便于故障定位

## 4. 高级功能

### 4.1 模型使用跟踪

StarLocalRAG 实现了模型使用跟踪机制，防止模型过早卸载，确保模型在需要时保持加载状态。当应用检测到模型不再使用时，会自动释放资源，优化内存使用。

### 4.2 文本转笔记

在RAG问答页面，可以通过长按选择AI回答文本，然后悬浮菜单选择"转笔记"，将有用的回答内容快速保存到知识库中。

### 4.3 模型选择记忆

应用支持"记住选择"功能，当选择嵌入模型时，可以选择将此选择保存到知识库元数据中，避免每次都需要重新选择模型。

### 4.4 分块大小选择

在设置中可以配置文本分块和重叠大小，提供多种预设选项：
- 4000/800（大块/大重叠，适合复杂文档）
- 2000/400（中大块/中重叠）
- 1000/200（默认，平衡选项）
- 500/100（小块/小重叠，适合简单文档）
- 100/20（微块/微重叠，适合精确检索）

## 5. 故障排除

### 5.1 常见问题

- **应用崩溃**：检查是否已授予存储权限，确保模型和知识库路径正确
- **模型加载失败**：确认模型文件格式正确，支持ONNX和TorchScript格式
- **知识库构建失败**：检查文件格式是否支持，尝试减小分块大小
- **回答质量不佳**：尝试调整系统提示词，或使用不同的分块大小

### 5.2 日志分析

应用提供详细的日志记录，可以通过"查看日志"菜单查看。日志包含四个级别：
- **DEBUG**：详细调试信息
- **INFO**：一般信息
- **WARNING**：警告信息
- **ERROR**：错误信息

查看日志可以帮助定位问题原因，特别是在处理大文件或复杂文档时。

## 6. 最佳实践

- **合理设置分块大小**：根据文档类型和复杂度选择合适的分块大小
- **选择合适的嵌入模型**：较大的模型通常效果更好，但会消耗更多资源
- **优化系统提示词**：良好的系统提示词可以显著提高回答质量
- **定期更新知识库**：随着信息变化，及时更新知识库内容
- **使用知识笔记整理信息**：将重要信息整理为知识笔记，方便后续检索
